## ✨ О проекте

**Bug Tracker** — это лёгкое веб-приложение для отслеживания багов, созданное с нуля на Flask. Оно позволяет удобно управлять задачами, прикреплять скриншоты, фильтровать данные и даже экспортировать отчёты с диаграммами.

Проект идеально подходит для:
- 🎓 **Учебных целей** — изучение Flask, SQLite, работы с изображениями
- 🏠 **Локальных проектов** — быстрый старт без сложной настройки
- 🛠️ **Пет-проектов** — основа для доработки и улучшения

## 🚀 Ключевые возможности

### 📝 Управление багами
| Функция | Описание |
|---------|----------|
| **➕ Добавление** | Полная форма с описанием, шагами, ожидаемым и фактическим результатом |
| **📋 Поля** | Title, Description, Steps, Expected, Actual, Severity, Priority |
| **📎 Вложения** | Поддержка изображений (скриншоты багов) |
| **✏️ Редактирование** | Изменение любой информации о баге |
| **🗑️ Удаление** | Быстрое удаление записей |

### 🎛️ Dashboard (панель управления)
- 🔄 **Сортировка** — по ID, Title, Severity, Priority
- 🎯 **Фильтрация** — по Severity (Critical/Major/Minor) и Priority (High/Medium/Low)
- 🖼️ **Просмотр изображений** — в модальном окне по клику
- ⚡ **Быстрые действия** — кнопки Edit и Delete для каждой записи

### 📊 Экспорт отчётов
- 📄 **HTML-отчёт** — генерируется одной кнопкой
- 📈 **Диаграмма** — распределение багов по Severity
- 🔢 **Статистика** — общее количество багов
- 🖼️ **Встроенные изображения** — в формате base64 прямо в HTML

## ⚙️ Установка и запуск

### Предварительные требования
- Python 3.x установлен на вашем компьютере
- Базовое знакомство с командной строкой

### Пошаговая инструкция

#### 1️⃣ Клонируйте репозиторий
```bash
git clone <url-репозитория>
cd burga-tracker
```

#### 2️⃣ Установите зависимости
```
pip install flask matplotlib
```

#### 3️⃣ Запустите сервер

```
python app.py
```

#### 4️⃣ Откройте в браузере

Перейдите по адресу: http://127.0.0.1:5000

## 🗄️ База данных

Приложение использует SQLite (bugs.db) с одной таблицей:

### Таблица bugs

# Таблица `bugs`

| Поле | Тип | Описание |
| :--- | :--- | :--- |
| `id` | INTEGER | Первичный ключ, автоинкремент |
| `title` | TEXT | Заголовок бага |
| `description` | TEXT | Описание проблемы |
| `steps` | TEXT | Шаги воспроизведения |
| `expected` | TEXT | Ожидаемый результат |
| `actual` | TEXT | Фактический результат |
| `severity` | TEXT | Уровень критичности |
| `priority` | TEXT | Приоритет |
| `attachment` | BLOB | Изображение в бинарном формате |

**Важно:** Изображения сохраняются непосредственно в БД как BLOB-объекты.

## 🖼️ Работа с изображениями

### Процесс работы:
- Загрузка — изображение прикрепляется через форму
- Сохранение — конвертируется в BLOB и сохраняется в БД
- Отображение — на dashboard показывается миниатюра
- Просмотр — по клику открывается модальное окно
- Экспорт — изображения встраиваются в HTML как base64

## 📤 Экспорт отчёта

### Формат экспорта
Кнопка Export HTML генерирует файл bug_report.html со следующими элементами:

```
📊 bug_report.html
├── 📈 Диаграмма Severity Distribution
├── 📊 Статистика по багам
└── 📋 Таблица со всеми багами
    └── 🖼️ Встроенные изображения (base64)
```

## 🧩 Маршруты (Routes)

| Маршрут | Метод | Описание |
| :--- | :--- | :--- |
| `/` | GET | Форма добавления нового бага |
| `/submit` | POST | Сохранение бага в БД |
| `/dashboard` | GET | Просмотр списка багов |
| `/edit/<id>` | GET/POST | Редактирование бага |
| `/delete/<id>` | POST | Удаление бага |
| `/attachment/<id>` | GET | Получение изображения |
| `/export` | GET | Экспорт HTML-отчёта |
| `/clear` | POST | Очистка базы данных |

## ⚠️ Ограничения
### Важно знать:

- 🚫 Не для production — только разработка и обучение
- 🔧 Flask dev server — не подходит для высоких нагрузок
- 🔓 Нет авторизации — полный доступ без ограничений
- 📦 Размер файла — не ограничен (нужно добавить вручную)
- 🛡️ Безопасность — нет защиты от XSS/SQL injection
